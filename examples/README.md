# Examples

These are **illustrative artifacts**, not actual run output — hand-written
samples that show the *shape* of what the team produces, so you can see how a
pipeline run reads end to end before you try it yourself.

Both docs below describe the **same** small change — *adding input validation
to a CSV-row parser* — so they read as one coherent run:

1. [`architect-design-doc.md`](architect-design-doc.md) — what `local-architect`
   hands back before any code is written (problem, approach, interface, affected
   files, risks).
2. [`review-synthesis.md`](review-synthesis.md) — what the lead produces after
   fanning out the five reviewers: a single consolidated synthesis, deduped and
   grouped by severity, attributed to reviewers, with `file:line` references and
   a follow-up-suggestions section.
